2、对你实验室研究的可信计算不是很了解，能讲一下吗？那你为什么默认可信根就是可信的？

答：

首先，我认为没有绝对的的安全，所以也就没有绝对的的可信。这个问题就类似于为什么ca是可信任的，杀毒软件是安全的。所以我们肯定要通过一些手段保证可信根的安全，在这个前提下，认为他是安全的。举个例子，在与华为合作的这个项目中，可信启动方案中一般通过一次性编程实现可信根，也就是将最底层的代码一次性编程在固件中，保证还可信根不被修改，然后保证出厂时默认安全的。这样一来该可信根就可以实现对上层代码的一个度量校验。

3、Linux使用的多吗？如何修改一个文件？除了使用VIM，还有其他方式吗？

（我写了Linux，所以才问的）

我回答了两点：

  • 可以直接使用编辑的方式，使用vi或者vim命令对文件进行一个修改。
  • 可以使用Linux重定向，（> 直接覆盖， >> 追加）
    • 使用 cmd < file，可以将文件作为命令的输入
    • 使用 cmd > file，可以将命令的执行结果输出到文件中
    • 使用管道符将命令的执行结果作为下一个命令的执行输入
    • 举例：head -n 10 file >> file

4、介绍一下HTTPS数据是如何实现加密的？

这个问题和（介绍HTTPS实现安全的原理是一样的），面试经常问。这里使用演绎的方法进行一个回答。

回答：

  • 首先，HTTP在传输数据时是明文传输的，是不安全的，因此在传输时，需要对数据进行一个数据加密。
  • 因为对称加密效率高，所以使用对称加密对数据进行加密。
  • 但是对称加密密钥总不能使用一个固定不变的进行传输。
  • 所以使用非对称加密对对称加密密钥进行一个加密。
  • 但是服务端在发布公私钥对的时，也就是将公钥传输至客户端时，存在中间人攻击
  • 讲解中间人攻击：客户端发送HTTTPS请求，服务端生成公私钥对，将公钥传给客户端，中间人劫持到公钥。然后发布自己的公私钥对，客户端拿到中间人的公钥，加密对称加密密钥，中间人使用私钥进行解密。窃听到对称加密密钥，并将密钥使用服务端的公钥加密发给服务端，
  • 为了解决中间人攻击，使用数字签名技术解决。
    • 首先CA生成公私钥对，发布自己的公钥，服务端使用CA的公钥，对服务端的公钥进行一个数字签名，服务端将在HTTPS过程中，服务端将数字证书传输给客户端，客户端收到数字证书后，使用CA的公钥进行验签，验签成功，即可以使用服务端的公钥对对称加密密钥进行加密传输。
    • 之所以客户端可以使用CA的公钥进行验签，是因为CA事先余留了CA的公钥。

5、介绍一下进程和线程的区别？进程有哪几种状态？

这个问题，面试官老喜欢问了，遇到很多次了，这次还是回答不太好（记忆的不是很清楚，回答的不够流畅，也不是很自信）。下面进行一个总结把。

总结：

(1) 你能介绍一下进程和线程的区别吗？

回答：

  • 好的，面试官。我先介绍一下什么是进程，什么是线程。
  • 进程是程序的一次执行过程，是系统运行程序的基本单位，因此进程是动态的。系统运行一个程序即是一个进程从创建、运行到消亡的过程。
  • 线程和进程相似，但是线程是一个比进程更小的执行单位。一个进程在执行过程中可以创建多个线程。
  • 我从JAVA角度介绍一下进程和线程之间的关系吧：
    • 一个进程可以有多个线程，多个线程共享进程的堆和方法区（JDK1.8之后是元空间）资源。但是每个线程有自己的程序计数器、虚拟机栈 和 本地方法栈。
    • Java中运行Main方法，JVM就会启动一个进程，main方法所在的线程为主线程。

总结来说：线程是进程表划分成的更小的运行单位，一个进程在其执行过程中可以产生多个线程。线程和进程最大的区别就是，个进程是独立的，而线程并不一定，因为同一进程中的线程是有可能相互影响的。线程的开销小，但是不利于资源的管理和保护；进程正好相反。

（2）进程的几种状态

回答：进程一般分为五种状态。

创建状态：进程正在被创建，还没进入就绪状态

就绪状态：进程已经处于就绪队列中，也就是进程获得了除了CPU以外的所有资源，一但获得了CPU即可运行

运行状态：进程正在CPU上运行

阻塞状态：又称为等待状态，进程正在等待某一资源或者等待IO操作，即使CPU空闲也不运行

结束状态：进程从系统中消失，可能是正常结束，也可能是异常退出。
